# Passport & Access

## The Simple Rule

People should be able to join the Discord server first.

The `KMH Passport` should not be a hard wall.

It should be the thing that gives people **more layer**, not basic existence.

---

## Recommended Flow

1. Join Discord
2. Read `#start-here`
3. Pick your roles or sphere
4. Fill the `KMH Passport` if you want more matching, visibility, and focused access
5. Move into `Academy`, `MVP Lab`, or `Networking`

---

## What Filling The Passport Can Unlock

- better path routing
- matching visibility
- approved member cards
- founder / MVP circles
- project and collaboration opportunities
- opt-in spotlighting

The point is:

`more value, not more bureaucracy`

---

## Best Access Model

Do **not** give raw private database access to everyone who submits a form.

Use three layers:

### `Layer 1` · Private Responses

Store:

- email
- Discord
- links
- interests
- goals
- notes

This stays private.

### `Layer 2` · Approved Member Cards

Show only what is safe and useful:

- name / nickname
- sphere
- looking for
- skills
- short idea or direction
- public links

This is the right layer for approved members.

### `Layer 3` · Public Registry

Use this for:

- startups
- experts
- SMB
- young talents
- allies

This is not the same as the private member database.

---

## Best Form Logic

Use **one main Passport form**, not three separate forms.

Main field:

- `Primary Goal`
  - `Academy`
  - `MVP / Build`
  - `Networking`
  - `Just Exploring`

Then use conditional sections:

- `Academy` -> path and learning block
- `MVP / Build` -> founder / idea / validation block
- `Networking` -> matching and collaboration block

---

## Consent Logic

Do not use one vague sentence.

Use separate checkboxes:

### Required

- `I agree that K Mentorship Hub may store my submission for community routing and matching.`

### Optional

- `I allow K Mentorship Hub to create a member card visible to approved community members.`
- `I allow K Mentorship Hub to mention my profile in team-building, matching, or MVP opportunities.`
- `I allow my public links and short profile to appear in community-facing spotlights or directories.`

---

## Anti-Spam Without Overengineering

Do not rely on one “magic question.”

Use a stack:

- Discord verification
- Discord AutoMod
- one short logic question in the form
- one intent question about `S / E / T`
- manual review for elevated roles

Good example questions:

- `Which sphere fits you best right now: S, E, or T, and why?`
- `What do you want first from the hub: learn, build, or connect?`
- `Name one direction you are actually interested in right now.`

These are not perfect anti-bot filters, but they are better than random puzzles.

---

## Best Practical Rule

Join should be easy.

Elevated access should be earned through:

- clearer intent
- a real submission
- opt-in visibility
- review before sensitive access

That keeps the hub open without giving everything away to low-quality signups.
